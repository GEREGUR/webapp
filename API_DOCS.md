# Документация API (Frontend)

## ℹ️ Общая информация

- **Base URL:** `/api/v1`
- **Авторизация:** Передача `initData` от Telegram в заголовках или параметрах (зависит от настройки `get_user`).

---

## 👤 Профиль и Кошелек

### Получение данных профиля

`GET /user/me`
Возвращает основную информацию о пользователе и его балансах.

- **Балансы:**
  - `internal_balance`: Баланс BP (внутренняя валюта).
  - `ton_balance`: Баланс TON.
- **Кошелек:**
  - `wallet_address`: Привязанный адрес (может быть `null`).
- **Рефералы:**
  - `referral_earn`: Сколько всего TON заработано с рефералов.

### Управление кошельком

- `POST /user/wallet/{address}` — Привязать адрес (валидация: 48 символов).
- `POST /user/wallet/clear` — Отвязать текущий кошелек.

### История транзакций

`GET /user/wallet/history?offset=0&limit=10`
Список всех изменений баланса (депозиты, покупки, награды).

- `type`: Тип операции (`CREATE_ORDER`, `BUY_ORDER`, `REFERRAL`, `PAYMENT`, `WITHDRAWAL`, `BATTLE_PASS`).
- `currency`: Валюта (`BP` или `TON`).
- `value`: Сумма изменения (положительная или отрицательная).
- `obj_id`: ID связанного объекта (например, ID ордера).

## 🎯 Задачи (Tasks)

### Список задач

`GET /user/tasks/list`
Возвращает все доступные задачи и текущий прогресс по ним.

- `goal_count`: Сколько нужно сделать для завершения.
- `progress`: Объект с текущим состоянием:
  - `current_count`: Текущий прогресс.
  - `status`: Статус (`ACTIVE` — в процессе, `COMPLETED` — выполнена, можно забрать награду, `REWARDED` — награда уже получена).
- **Награды:** Указаны в полях `reward_ton`, `reward_bp`, `reward_exp`.

### Действия с задачами

- `POST /user/tasks/activate/{id}` — Начать выполнение задачи.
- `POST /user/tasks/completed/{id}` — Отправить на проверку (только для задач типа Story, Bio, Nick).
- `POST /user/tasks/claim/{id}` — Забрать награду после выполнения.

## ⚔️ Battle Pass

### Состояние BP

`GET /battlepass/me`

- `level`: Текущий уровень.
- `exp`: Текущий опыт.
- `progress`: Процент прогресса до следующего уровня (0-100).
- `rewards`: Список наград по уровням.
  - `is_available`: Можно ли забрать (зависит от уровня).
  - `is_claimed`: Уже получена.

### Получение награды

`POST /battlepass/claim/{reward_id}` — Забрать награду за достигнутый уровень.

## 📈 Маркетплейс (Ордера)

### Создание ордера

`POST /order/create`
Выставить BP на продажу.

- **Body:** `{ "bp_amount": 1000 }`
- **Логика:** Списывает BP, создает ордер. TON зачислится по мере выкупа ордера другими игроками.

### Покупка (Выкуп ордера)

`POST /order/buy`
Купить BP за TON у другого пользователя.

- **Body:** `{ "order_id": 1, "ton_amount": 5.5 }`
- **Логика:** Можно выкупить ордер частично. Минимальная сумма покупки ограничена настройками.

### Отмена (Self Buy)

`POST /order/self_buy/{order_id}`
Закрыть свой ордер и вернуть остаток средств.

- **Логика:** Возвращает TON (за вычетом комиссии системы) на баланс пользователя.

### Информация об ордере

`GET /order/info/{id}`
Детальные данные перед покупкой: лимиты (min/max buy), сколько BP будет получено в качестве бонуса.

## 💰 Депозит и Вывод

### Пополнение (Deposit)

1. Вызвать `GET /wallet/payment/data`.
2. Получить `address` (куда слать) и `memo` (комментарий).
3. **Важно:** Пользователь должен отправить TON с указанным `memo`, иначе платеж не зачислится автоматически.
4. После отправки можно вызвать `POST /wallet/payment/transaction?memo=...` для ускорения проверки.

### Вывод (Withdrawal)

`POST /wallet/withdrawal`

- **Body:** `{ "address": "...", "amount": 10.0 }`
- **Логика:** Списывает TON с баланса и создает заявку на вывод.

## 🔌 WebSocket (Real-time)

**URL:** `wss://domain.tld/api/v1/socket/ws?auth=user_id`

Сервер использует WebSocket для трансляции изменений на рынке и уведомлений пользователя. Все сообщения приходят в формате JSON и имеют общую структуру:

```json
{
  "type": "название_события",
  "data": { ... объект с данными ... }
}
```

### 📥 Исходящие события (от сервера к клиенту)

#### 1. `initial_state`

Отправляется один раз сразу после успешного подключения. Содержит текущий срез данных для инициализации интерфейса.

- **Структура `data`:**
  - `stats`: `{ "total_ton": float, "total_orders": int }` — Общая статистика.
  - `orders`: `Array<Order>` — Список из 50 самых актуальных активных ордеров.
  - `history`: `Array<Deal>` — Список из 10 последних совершенных сделок.

#### 2. `new_order`

Приходит, когда любой пользователь создает новый ордер.

- **Структура `data` (Объект `Order`):**
  - `id`: `int` — ID ордера.
  - `owner`: `{ "id": int, "avatar": string, "name": string, "username": string }` — Данные создателя.
  - `initial_bp_amount`: `float` — Сколько BP выставлено изначально.
  - `initial_ton_amount`: `float` — Общая стоимость в TON.
  - `current_ton_amount`: `float` — Сколько TON осталось выкупить.
  - `status`: `string` (`OPEN`, `PARTIAL`).
  - `create_date`: `int` — Timestamp создания.

#### 3. `order_update`

Приходит, когда ордер частично выкуплен или закрыт.

- **Структура `data`:**
  - `id`: `int` — ID ордера.
  - `current_ton_amount`: `float` — Новый остаток в TON.
  - `status`: `string` (`PARTIAL`, `CLOSED`).

#### 4. `new_deal`

Уведомление о факте совершения сделки (кто-то купил BP).

- **Структура `data`:**
  - `order_id`: `int` — ID ордера, по которому прошла сделка.
  - `ton_amount`: `float` — Сумма конкретно этой сделки в TON.

#### 5. `stats_update`

Обновление глобальных показателей рынка.

- **Структура `data`:**
  - `total_ton`: `float` — Общий объем TON в системе.
  - `total_orders`: `int` — Общее количество активных ордеров.

#### 6. `orders_bump`

Приходит, когда пользователь "поднимает" свои ордера в списке.

- **Структура `data`:**
  - `orders`: `Array<Order>` — Список обновленных ордеров с актуальными датами.

#### 7. `success_payment` (Персональное)

Приходит только тому пользователю, чей депозит был успешно зачислен.

- **Структура `data`:**
  - `amount`: `float` — Сумма зачисления в TON.
  - `tx_hash`: `string` — Хеш транзакции в блокчейне.

#### 8. `success_withdrawal` (Персональное)

Приходит только пользователю, чей вывод средств был успешно обработан.

- **Структура `data`:**
  - `amount`: `float` — Сумма вывода.

### 📤 Входящие события (от клиента к серверу)

#### `update_filter`

Позволяет клиенту ограничить поток событий `new_order`. Если фильтр установлен, сервер будет присылать уведомления только о тех новых ордерах, чей объем в TON больше или равен указанному.

- **Формат:**

```json
{
  "type": "update_filter",
  "min_ton": 1.5
}
```

- `min_ton`: `float` — Минимальный порог. Чтобы сбросить фильтр, отправьте `0`.
